# Social Copy Patterns

这份文档沉淀中文社媒文案常用结构。使用时要根据材料自适应，不要机械套模板。

## 最高原则

- 开头先抓住读者，不要先解释背景。
- 抓人必须靠具体事实，不靠空泛夸张。
- 禁止照搬参考帖的固定句式和口头禅。
- 输出要像真人分享，不像 AI 总结。

## 抓人但可信的开头

当文案显得太克制时，不要加“封神、爆款、全网最”。要把刺激点换成更具体的利益、风险、结果。

可用骨架：

- `想跟 {项目/赛道} 跑得更近，{入口/工具} 值得先盯。`
- `{对象} 里真正值钱的，是 {具体信号/结果/反馈}。`
- `{场景} 卡住时，先看 {入口/方法}，能少踩 {具体坑}。`
- `只看 {公开信息源}，容易漏掉 {社区/issue/会议} 里的早期反馈。`
- `{工具/项目} 的早期坑，往往会先在 {社区/issue/讨论} 里露出来。`
- `想判断 {项目/团队} 是否重视 {人群}，看 {具体反馈场景} 更直接。`

示例：

- `想跟 Arc 生态跑得更近，Arc House 比公告更值得盯。`
- `Arc House 里真正值钱的，是 builder 和团队互动留下的早期信号。`
- `Agent Wallet 在 Windows 上卡住时，先看社区里的安装反馈和修复节奏。`

## 禁用模板

默认不要使用：

- 很多人 / 大家都 / 大多数人
- 别只当成 / 别只把 / 别再以为
- 不是……而是……
- 不是普通 X，更像 Y
- X 不在……而在……
- 简单讲 / 简单理解 / 简单说
- 没证据的 99% / 全网最 / 千万不要

替换方式：

- 把泛泛人群换成具体人群。
- 把抽象判断换成具体结果。
- 把解释型开头换成使用场景。
- 把夸张词换成可验证事实。

## 标准帖子结构

### 1. 开头

1-2 句完成：受众、主题、收益或风险。

例：

`想让 Codex 从“能写代码”升级到“能接管开发工作流”，插件别乱装，先补这几类高频入口。`

### 2. 定位

用一句话交代对象是什么。

例：

`Arc House 是 Arc 给 builders 准备的社区入口，承接 Office Hour、项目交流、活动和资源分享。`

### 3. 重点

每个重点先给判断，再解释。

结构：

`1. 判断句`

`解释 1-2 句，补充具体事实。`

### 4. 收尾

收尾给出适合人群、下一步动作或判断标准。

例：

`如果你关注 Arc 的 AI Agent 支付线，Arc House 可以先加入信息源。这里适合观察早期反馈、工具坑和团队响应节奏。`

## 常用角度

### 工具推荐

- 先说适合谁。
- 再说解决什么高频问题。
- 最后说按什么组合安装。

骨架：

`如果你用 Codex 写真实项目，先装能接住开发工作流的工具：仓库、文档、浏览器验证、代码检索。`

### 项目解读

- 先说为什么值得关注。
- 再说项目入口和核心能力。
- 最后说风险、限制或适合人群。

骨架：

`关注 {项目} 的人，{入口/工具} 值得先看，因为它能看到 {具体信号}。`

### 社区/生态观察

- 先说社区里能看到什么真实信号。
- 再说活动、资源、团队互动。
- 最后说如何判断是否值得长期关注。

骨架：

`判断一个生态是否重视 builders，看问题有没有被接住、资源有没有集中、活动有没有持续。`

### 避坑/问题反馈

- 先点具体坑。
- 再说影响谁。
- 最后说怎么处理或观察。

骨架：

`{工具} 在 {环境} 上卡住，先看 {具体问题}。这类早期坑本身不大，但能看出团队修复速度。`

## 语气控制

更像真人：

- `这个入口值得盯。`
- `这里能看到早期反馈。`
- `这个坑不大，但能看团队响应。`
- `如果你在看这条线，可以先收藏。`

更像 AI，少用：

- `该平台旨在赋能全球开发者生态。`
- `这体现了其在未来智能经济中的重要战略意义。`
- `从底层逻辑来看，它重塑了价值流转范式。`

## 检查清单

发布前检查：

- 第一行是否抓住具体读者。
- 是否给出具体收益、风险或结果。
- 是否避开禁用模板。
- 是否兼顾吸引力和解释清晰度。
- 是否每个强判断都有事实承接。
