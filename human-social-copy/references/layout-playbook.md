# 图文排版参考

用于把一篇中文社交媒体内容整理成更容易阅读、收藏和转发的图文结构。适合公众号、小红书、X长文、Threads长文和信息图脚本。

核心原则：内容先抓住人，再让读者顺着读完，最后给一个轻动作。

## 黄金结构

一篇图文内容优先使用这个结构：

1. 标题
2. 开头留人
3. 正文骨架
4. 结尾互动

### 标题

标题是流量入口，最好在3秒内让读者知道这篇内容和自己有关。

标题可以用：

- 痛点：`新手看美股，最容易错在这一步`
- 解决方案：`这周美股只看3个信号就够了`
- 数字清单：`财报季先看5个位置`
- 反常识：`财报好，股价也可能跌`
- 风险提醒：`AI芯片再热，也别只看新闻热度`

标题不要只写大词，比如：

- `市场观察`
- `行业分析`
- `趋势解读`
- `深度思考`

这类标题没有明确读者，也没有明确收益。

### 开头留人

开头不要绕太远，前3到5行直接进入读者关心的问题。

常用方式：

- 先指出一个误区
- 先说一个真实困扰
- 先给本篇能帮读者解决什么
- 先用一句话圈定读者

示例：

`美股最容易让新手犯错的时候，不是没消息，而是消息太多。`

`这周如果你只看单日涨跌，很容易被带着跑。`

`财报季别只看公司赚没赚钱，市场更关心的是有没有超预期。`

### 正文骨架

正文控制在3到5个重点。每个重点最好只承担一个任务。

常见结构：

- 这是什么
- 为什么重要
- 新手容易看错哪里
- 真正应该看什么
- 下一步观察哪个信号

如果内容比较复杂，可以用小标题。但小标题要短，最好6到12个字。

好的小标题：

- `先看Fed纪要`
- `再看财报指引`
- `最后看AI芯片`

少用抽象小标题：

- `核心逻辑`
- `深度思考`
- `重要原因`
- `底层判断`

### 结尾互动

结尾不要用强销售语气。

更适合用轻动作：

- `先收藏，周中看盘前对照一遍。`
- `这周你最关注Fed、财报，还是AI芯片？`
- `如果你也在看美股，可以把这几个信号放进观察清单。`

不要用：

- `赶紧收藏`
- `一定要转发`
- `错过就亏大了`
- `最后机会`

## 版式规则

适合中文社交媒体的排版：

- 每段尽量1到3行
- 一个段落只表达一个意思
- 长句拆短，不要一口气塞太多信息
- 关键句单独成段
- 列表控制在3到5项
- 小标题不要太多
- 不要连续堆大段解释

如果是小红书或公众号图文，首屏要优先放：标题、读者痛点、3个核心点。

如果是X或Threads，第一屏要优先放：钩子、核心结论、读者能获得什么。

## 信息图建议

配图不要只放装饰图。最好让图片承担信息增量。

适合配图：

- 3个核心信号卡片
- 观察清单
- 对比表
- 简单流程图
- 关键指标监控图

金融内容可以配：

- Fed预期变化示意图
- 财报观察清单
- AI资本开支监控图
- 芯片产业链分层图
- 风险信号列表

不建议配：

- 纯AI机器人图
- 夸张爆炸图
- 随机芯片照片
- 没有来源的假数据图
- 信息量过低的装饰背景

## 图文风格

金融、AI、crypto内容建议统一成克制、清楚、有可信度的视觉风格。

推荐：

- 深色背景配白色文字
- 关键数字用单一强调色
- 图表线条清楚，不要堆特效
- 保留足够留白
- 字号层级清楚

避免：

- 颜色超过三种
- 字太小
- 一张图塞太多信息
- 标题党视觉
- 低清素材

## 最终检查

发布前检查：

- 标题有没有具体读者或具体收益
- 开头3秒内有没有进入主题
- 正文是否控制在3到5个重点
- 每个重点有没有具体信号或动作
- 结尾是不是轻互动
- 配图有没有提供信息增量
- 排版有没有留白
