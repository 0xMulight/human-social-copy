# Adaptive Opening Hooks

这份文档用于把素材自适应改写成更适合中文社媒的开头钩子。重点不是套爆款句式，而是根据内容本身找到最能让读者停下来的入口。

## 总原则

钩子允许有冲突感、结果感、反差感、风险感，但不能靠空泛夸张撑场面。

第一句必须让人知道：

- 这篇内容和谁有关。
- 能解决什么问题，或能看到什么信号。
- 为什么值得继续看。

克制不等于低刺激。默认要写到“抓人但可信”，不要停在说明文口吻。

## 钩子强度三档

同一份材料先生成三档，再选择最适合发布的一档。

### 1. 稳重版

用于解释型内容、行业观察、资料整理。

例：

- `Arc House 是 Arc builders 的社区入口，适合长期关注生态的人先看。`
- `Agent Wallet 是 Arc 面向 AI Agent 支付场景的一块关键工具。`

### 2. 抓人版

默认优先使用。它要比稳重版更有结果感、冲突感或场景感。

例：

- `想跟 Arc 生态跑得更近，Arc House 比公告更值得盯。`
- `Arc House 里真正值钱的，是 builder 和团队互动留下的早期信号。`
- `Agent Wallet 卡在 Windows 安装时，先看社区反馈和团队修复节奏。`

### 3. 强钩子版

用于竞争激烈的信息流。可以更尖，但必须被正文事实支撑。

例：

- `只看官推，很容易漏掉 Arc 生态里更早冒出来的问题和机会。`
- `Agent Wallet 的早期坑，可能会先在 Arc House 这类 builder 社区里暴露。`
- `想判断 Arc 团队是不是真的重视 builders，看社区里的问题怎么被处理就够直接。`

## 强度来源

增强钩子时，不要加空泛形容词。优先加这四类信息：

- 具体人群：`正在装 Agent Wallet 的开发者`、`关注 Arc 的 builder`。
- 具体结果：`更早看到 bug`、`判断团队响应速度`、`少踩安装坑`。
- 具体冲突：`公告很干净，社区反馈更接近真实使用现场`。
- 具体场景：`Office Hour`、`Windows npm module 缺失`、`community.arc.network`。

可用词：

- `真正值钱的`
- `值得盯`
- `先收藏`
- `卡住先看`
- `更早看到反馈`
- `真实使用现场`
- `早期信号`

这些词必须绑定具体对象。不要写成“这个太强了”“直接封神”。

## 开头禁区与改写法

默认避免这些句式：

- `很多人/大家都/大多数人……`
- `别只当成/别只把/别再以为……`
- `不是……而是……`
- `不是普通 X，更像 Y`
- `X 不在……而在……`
- `简单讲/简单理解/简单说……`
- 没有证据的 `99%的人/全网最/千万不要`

改写示例：

- 弱：`很多人把 Arc House 当成普通社区，其实不是。`
- 强：`Arc House 里真正值钱的，是 builder 和团队互动留下的早期信号。`
- 弱：`别只把 Agent Wallet 当成普通钱包。`
- 强：`Agent Wallet 的重点，是让 AI Agent 在受控规则下完成 USDC 支付和交易。`
- 弱：`简单讲，Arc House 就是 Arc 的 Discord。`
- 强：`Arc House 更像 Arc 给 builders 准备的社区入口，承接 Office Hour、项目交流和生态资源。`

## 8 个核心钩子模型

### 1. 好奇模型

公式：`只说一半的关键发现 + 留白`

适合：教程、工具、清单、经验总结。

例：

- `我发现 Codex 真正提效的关键，不是多装插件。`
- `Arc House 里有个信号，比活动本身更值得看。`

注意：留白后面必须马上补信息，不要故弄玄虚。

### 2. 痛点模型

公式：`精准扎心困境 + 立刻给路径`

适合：教程、避坑、工具推荐、效率方法。

例：

- `MCP 装完没效果时，先检查权限和调用场景。`
- `Agent Wallet 在 Windows 上部署卡住，先看 npm module 缺失这类已暴露的问题。`

### 3. 反差模型

公式：`过去糟糕状态 + 现在可验证结果`

适合：经验复盘、工具对比、成长故事。

例：

- `以前让 AI 改文案只会变油，现在先给它钩子规则，效果会稳很多。`
- `同样是社区入口，Arc House 更值得看的地方在 builder 互动和问题反馈。`

### 4. 圈定受众模型

公式：`点名具体人群 + 直接说明收益`

适合：垂直内容、工具推荐、行业帖。

例：

- `刚开始用 Codex 的人，先从 GitHub 和 Context7 这类高频工具补起。`
- `关注 Arc 的 builder，可以先把 Arc House 加进信息源。`

### 5. 风险提醒模型

公式：`指出潜在风险 + 给缓解方法`

适合：安全、投资、职业、工具避坑。

例：

- `只看项目公告，容易错过社区里先暴露的使用问题。`
- `Agent Wallet 这种早期工具，先观察安装坑和修复速度，再判断投入节奏。`

### 6. 后果模型

公式：`放大坏结果 + 给预防方法`

适合：避坑、决策、成长、工具选择。

例：

- `插件装太多，Codex 反而会变慢；先按工作流缺口补。`
- `早期生态工具链处理太慢，builder 很容易流失；Arc House 的反馈节奏值得观察。`

### 7. 高价值模型

公式：`别处没有的独家干货/稀缺信息`

适合：资源合集、独家观察、教程。

例：

- `Codex 插件先按工作流缺口装，这套组合可以先收藏。`
- `Arc House 里真正值得收藏的，是 Office Hour、builder 反馈和团队回应放在同一个入口。`

### 8. 借势模型

公式：`明星/专家/官方/热门事件 + 你的判断`

适合：热点追踪、行业观察、项目更新。

例：

- `Circle 推 Arc 后，最值得观察的一条线是 AI Agent 支付怎么落地。`
- `围绕 Arc 的 Agent Wallet，社区里的安装反馈比宣传词更能说明成熟度。`

## 补充钩子模型

### 数字清单模型

公式：`具体数字 + 明确收益`

例：

- `Codex 想接管开发工作流，先装这 5 类工具。`
- `看 Arc House 这类 builder 社区，先盯 3 个信号。`

### 省时省钱模型

公式：`成本痛点 + 省下的东西`

例：

- `别把时间花在低频插件上，Codex 先补这几个高频入口。`
- `跟 Arc 生态信息时，Arc House 能帮你少翻好几个分散入口。`

### 新手友好模型

公式：`新手误区 + 正确路径`

例：

- `新手用 Codex，先别急着装一堆 MCP。`
- `刚看 Arc 的人，先从 Arc House 了解 builder 活动和工具反馈。`

### 亲测体验模型

公式：`个人体验 + 具体结果`

例：

- `我试过之后发现，Codex 真正常用的插件其实很集中。`
- `看完 Arc House 的信息后，我会先盯 Agent Wallet 的安装反馈和团队修复。`

### 误区纠偏模型

公式：`常见错误判断 + 更具体的判断`

例：

- `评价一个 Agent 工具，别只看星标；看它能不能进你的真实工作流。`
- `判断 Arc House 的价值，别只看社区人数；看 builder 问题有没有被认真回应。`

## 组合使用

一个标题可以融合 2 种以上钩子，但不能堆太满。

好组合：

- `圈定受众 + 结果感`
- `痛点 + 解决方案`
- `风险提醒 + 具体动作`
- `好奇 + 高价值`

例：

- `关注 Arc 的 builder，Arc House 这 3 类信息值得先盯。`
- `Agent Wallet 在 Windows 上卡住时，先看这类已暴露的安装问题。`

## 黄金位置

最强钩子优先放在：

- 标题前 10 个字。
- 正文第一句。
- 小标题开头。

如果第一句只是背景介绍，读者很容易滑走。

## 输出建议

给用户输出时，可以提供：

- 1 个推荐开头。
- 2-4 个备选开头。
- 每个开头标注属于哪种钩子模型。

默认不要只给稳重版；至少给一个“抓人但可信”的版本。
