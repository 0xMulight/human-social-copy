# 真人感社媒文案Skill

这是一个用于中文社媒文案写作的CodexSkill，重点是把草稿、笔记、截图内容、项目资料或粗略想法，改成更像真人写的社媒内容。

它不是用来写华丽宣传稿的，而是用来写更朴素、更清楚、更可信的中文内容。

## 这个仓库能做什么

它可以帮助AI在写中文社媒内容时稳定做到：

- 按“钩子加干货加CTA”的结构输出
- 第一行先抓住读者注意力，但不夸张
- 中间给真实经验、步骤、清单、判断标准或注意事项
- 结尾只引导一个动作，比如评论、收藏、关注、分享、尝试、提问
- 去掉明显AI腔、空话、套话和过度修饰
- 不写线程，除非用户明确要求
- 清理括号和括号里的补充说明
- 减少分类小标题，让内容更像自然分享
- 避免“大家以为”这类老套开头
- 避免“不是……而是……”这种排比感很重的句式
- 少用“当”字开头的句子
- 删除“旨在、赋能、打造、范式、这种、硬生生、扒、助力、路径、逻辑、痛点、说白了、护城河”等常见AI词
- 去除英文单词或缩写前后的空格，比如把“AI 腔”改成“AI腔”，把“加 CTA”改成“加CTA”

## 适合哪些内容

这个Skill适合这些场景：

- X帖子
- Threads帖子
- Instagram文案
- TikTok口播稿
- cryptoYap内容
- 空投解析
- 项目观察
- 工具教程
- AI工具体验
- AI模型更新解读
- AI工作流分享
- 个人经验复盘
- 社群公告改写
- 把AI草稿改得更像真人表达

## 选题规则

写社媒内容时，优先看两点：

1. 内容要对读者有用

教程、工具、清单、避坑、案例、数据整理，通常比泛泛观点更容易被读者看完。

2. 选题要贴近当下热度

可以结合市场正在讨论的话题、项目更新、空投规则变化、任务变化、代币消息、AI模型更新、AI工具发布、AI新玩法，或者社区里反复出现的问题。

## 使用方式

把`human-social-copy`文件夹放到Codexskills目录后，可以这样调用：

```text
使用$human-social-copy，把这段草稿改成钩子加干货加CTA的中文社媒文案。
```

## 仓库结构

```text
human-social-copy/
  SKILL.md
  agents/openai.yaml
  references/patterns.md
```

## 文件说明

`human-social-copy/SKILL.md`是核心规则，决定AI什么时候使用这个Skill，以及写作时必须遵守哪些要求。

`human-social-copy/agents/openai.yaml`是技能展示信息，包含名称、简介和默认调用提示。

`human-social-copy/references/patterns.md`是参考模板，放了常用开头、基础公式、cryptoYap文案结构和改写检查清单。

`CONTRIBUTING.md`是贡献说明，方便别人知道怎么改这个Skill。

`LICENSE`是MIT开源协议。
