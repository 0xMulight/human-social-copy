# human-social-copy通用Agent规则

本仓库给各类Agent使用。处理中文社媒文案时，按下面规则写。

适用场景：X、Threads、Instagram、TikTok、小红书、cryptoYap、空投解析、项目观察、工具教程、AI工具体验、AI模型更新解读、AI工作流分享、个人复盘。

## 核心要求

输出要像真人在分享经验。语气朴素，信息具体，判断克制。

默认结构是钩子加干货加CTA：

1. 第一行先抓住重点，别夸张。
2. 中间给真实经验、步骤、清单、判断标准、限制或注意事项。
3. 结尾只引导一个动作，比如评论、收藏、关注、分享、尝试、提问。

最终文案里不要写出“钩子、干货、CTA”这些标签。

## 选题要求

优先选有用又贴近当下热度的内容。

有用内容包括教程、工具、清单、避坑、案例、数据整理、真实体验。

当下热度包括市场正在讨论的话题、项目更新、空投规则变化、任务变化、代币消息、AI模型更新、AI工具发布、AI新玩法、社区反复提到的问题。

写AI内容时，不要编模型能力、发布时间、价格、上下文长度、产品功能或官方承诺。需要最新事实时先核实；核实不了就提醒用户哪些信息需要确认。

写crypto或空投内容时，不要编数据、日期、奖励、合作、融资、规则。

## 语言规则

- 使用简体中文。
- 用户没有要求时，不要写成线程。
- 不保留括号和括号里的补充说明。
- 不要堆很多分类小标题。
- 不要总用“大家以为”开头。
- 少用“当”字开头的句子。
- 避免“不是……而是……”这类排比感很重的句式。
- 去掉夸张修饰、宏大比喻和绝对化表达。
- 多写具体经验、真实取舍、检查方法，少写口号。
- 中文和英文单词或缩写之间不要留空格，例如写成AI工具、GitHub项目、CTA结尾、crypto内容。

## 禁用词

最终文案里删除这些词，除非用户明确要求保留原文引用：

旨在、赋能、打造、范式、这种、硬生生、扒、助力、路径、逻辑、痛点、说白了、护城河

遇到类似空泛商务词，也换成更直接的说法。

## 输出前检查

- 第一行单独看也能抓住重点。
- 中间真的有信息，不是口号。
- 结尾只有一个CTA。
- 没有括号。
- 没有禁用词。
- 没有明显AI腔。
- 英文单词或缩写前后没有多余空格。
- 语气真实、克制、可信。
- 结论没有超过证据本身。
